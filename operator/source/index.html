SecureGate Guide

Protocol telemetry console — not a wallet replacement

Standard EOA rules always apply: private key = control. If a key leaked, rotate to a new wallet, move funds, and revoke approvals first. This console monitors lane coherence — it does not override Ethereum wallet semantics.
Ownership verification — this app belongs to Empress (@Hope_ology). The full design (logic, workflows, variables, standards choices) is solely hers, including LLM-assisted assembly. Use Ownership for the plain checklist. You verify you're the authorized operator, not claiming on-chain copyright.

1. Unlock & connect

Set your transport endpoint (RPC) — dashboard operates immediately; operator phrase is only for source export. Registry anchor auto-fills after unlock. Lane tokens are optional for read-only telemetry.

2. Daily use (most people)

  • Telemetry — lane health, mesh status, staged packets
  • Beacon — origin pulse + drift check
  • Deploy — fabric bootstrap + funding estimates
  • Trace — session activity log

3. Before lane changes

Bind session coherence (origin vector + epoch marker). Required for Queue, Authorize, Execute, and Sever. TTL: 30 minutes.

4. Lane map

α ingressActive wallet under observation — stages packets
β attestOffline corroboration — paste signatures only
γ terminusRegistry sink — receive path configured at bootstrap
CourierEphemeral fee wallet for relay-paid network costs

5. Session safety

Credentials are memory-only. Esc, idle timeout, or tab hide locks the shell. Use Purge Tokens before walking away. Default Unmask: OFF blurs addresses.
